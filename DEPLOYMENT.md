## Security

### Rules

- **Never** commit secrets, passwords, JWT keys, or API keys to Git
- **Never** put secrets in frontend env vars (only `VITE_*` public values)
- Use platform env var dashboards or GitHub Secrets for CI

### Generate JWT secret

```bash
openssl rand -base64 64
```

### GitHub Secrets (for CI/CD extensions)

If you extend CI to deploy automatically, store these in **GitHub ‚Üí Settings ‚Üí Secrets**:

| Secret | Used for |
|--------|----------|
| `JWT_SECRET` | CI test runs (already uses inline test value) |
| `RENDER_API_KEY` | Optional: automated Render deploys |
| `VERCEL_TOKEN` | Optional: automated Vercel deploys |

Current CI workflows do **not** require secrets ‚Äî they only build and test.

### Render secrets

Set in **Render Dashboard ‚Üí Service ‚Üí Environment**:

- `JWT_SECRET` ‚Äî auto-generated by `render.yaml` Blueprint
- `DATABASE_PASSWORD` ‚Äî auto-populated from linked PostgreSQL
- `CORS_ALLOWED_ORIGINS` ‚Äî your Vercel URL (manual, after frontend deploy)

### Vercel secrets

Set in **Vercel Dashboard ‚Üí Project ‚Üí Settings ‚Üí Environment Variables**:

- `VITE_API_BASE_URL` ‚Äî your Render API URL + `/api`

## Backend ‚Äî Render

### Option A: Blueprint (recommended)

1. Push code to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com) ‚Üí **New** ‚Üí **Blueprint**
3. Connect your GitHub repository
4. Render reads `render.yaml` and creates:
   - `finbrain-api` web service (Docker)
   - `finbrain-db` PostgreSQL database
5. Wait for first deploy (~5‚Äì10 minutes)
6. Note your API URL: `https://finbrain-api.onrender.com`

### Option B: Manual setup

1. **New ‚Üí PostgreSQL** ‚Üí name `finbrain-db` ‚Üí create
2. **New ‚Üí Web Service** ‚Üí connect GitHub repo
3. Settings:
   - **Root Directory:** `backend`
   - **Runtime:** Docker
   - **Dockerfile Path:** `./Dockerfile`
   - **Health Check Path:** `/actuator/health`
4. Add environment variables (see [Backend Variables](#backend-render-variables))
5. Link PostgreSQL database
6. Deploy

### Configure CORS (after frontend deploy)

In Render ‚Üí `finbrain-api` ‚Üí Environment:

```
CORS_ALLOWED_ORIGINS=https://your-app.vercel.app
FRONTEND_URL=https://your-app.vercel.app
```

Click **Manual Deploy** to apply.

### Verify backend

```bash
curl https://finbrain-api.onrender.com/actuator/health
# Expected: {"status":"UP"}

curl https://finbrain-api.onrender.com/api/health
# Expected: {"status":"ok"}
```

### Monitor logs

Render Dashboard ‚Üí `finbrain-api` ‚Üí **Logs**

Look for:
- `Started FinanceTrackerApplication`
- No `Connection refused` database errors
- No `JWT secret` startup failures

## Frontend ‚Äî Vercel

### Deploy

1. Go to [Vercel Dashboard](https://vercel.com/dashboard) ‚Üí **Add New Project**
2. Import your GitHub repository
3. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Add environment variable:

   ```
   VITE_API_BASE_URL=https://finbrain-api.onrender.com/api
   VITE_APP_NAME=Finbrain
   ```

5. Click **Deploy**

### Verify frontend

1. Open your Vercel URL
2. Splash screen ‚Üí login page
3. Register a new account
4. Dashboard loads with no CORS errors in browser DevTools
5. Deep link test: navigate to `/plan`, refresh ‚Äî page should load (SPA routing)

### Update CORS on Render

After you know your Vercel URL, update Render env vars and redeploy backend.

## Deployment Checklist

### Pre-deploy

- [ ] Code pushed to GitHub `main` branch
- [ ] `./mvnw clean package` succeeds locally
- [ ] `npm run build` succeeds locally
- [ ] `docker build -t finbrain-api ./backend` succeeds
- [ ] Strong `JWT_SECRET` generated (if not using Blueprint auto-gen)

### Backend (Render)

- [ ] PostgreSQL database created and linked
- [ ] Docker web service deployed
- [ ] `SPRING_PROFILES_ACTIVE=prod`
- [ ] All `DATABASE_*` variables populated
- [ ] `/actuator/health` returns `{"status":"UP"}`
- [ ] `/api/health` returns `{"status":"ok"}`
- [ ] Logs show no startup errors

### Frontend (Vercel)

- [ ] Root directory set to `frontend`
- [ ] `VITE_API_BASE_URL` points to Render API + `/api`
- [ ] Build succeeds on Vercel
- [ ] SPA routing works (refresh on `/plan`, `/transactions`, etc.)

### Integration

- [ ] `CORS_ALLOWED_ORIGINS` set to Vercel URL on Render
- [ ] Register + login works end-to-end
- [ ] JWT authentication persists across page navigation
- [ ] API calls return data (dashboard, transactions, etc.)
- [ ] No CORS errors in browser console
- [ ] HTTPS enabled on both URLs (automatic on Render/Vercel)

### Not applicable (current codebase)

- [ ] File uploads ‚Äî not implemented
- [ ] Email sending ‚Äî not implemented (env vars reserved for future)

### CI/CD

- [ ] GitHub Actions backend workflow passes
- [ ] GitHub Actions frontend workflow passes

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| `Application failed to start: JWT secret` | Missing `JWT_SECRET` in prod | Set in Render env vars |
| `Connection to DB refused` | PostgreSQL not linked | Link database in Render dashboard |
| CORS error in browser | Wrong origin | Set exact Vercel URL in `CORS_ALLOWED_ORIGINS` |
| 401 on every API call | Token signed with old secret | Clear localStorage, log in again |
| Vercel 404 on refresh | Missing SPA rewrite | Ensure `frontend/vercel.json` exists |
| API calls go to wrong host | Missing env var | Set `VITE_API_BASE_URL`, redeploy Vercel |
| Render health check fails | App still starting | Increase health check start period to 90s |
| Docker build fails | Maven dependency issue | Run `./mvnw clean package` locally first |
| Free tier cold start | Render spins down after inactivity | First request takes 30‚Äì60s ‚Äî normal on free tier |

## Docker Local Testing

```bash
# Build
docker build -t finbrain-api ./backend

# Run with dev profile (uses H2 ‚Äî for quick smoke test only)
docker run -p 8081:8081 \
  -e SPRING_PROFILES_ACTIVE=dev \
  -e JWT_SECRET=local-docker-test-secret-min-32-chars \
  -e PORT=8081 \
  finbrain-api

# Health check
curl http://localhost:8081/actuator/health
```

For production-like testing, connect to a real PostgreSQL instance via `DATABASE_*` env vars.
